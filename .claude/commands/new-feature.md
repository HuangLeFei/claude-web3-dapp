# 新功能开发流水线

触发完整的 DApp 功能开发流水线：PM 需求 → 架构设计 → 合约开发 → 安全审计 → 测试。

## 使用方式

```
/new-feature {功能描述}
```

## 示例

```
/new-feature 实现一个 ERC-20 代币质押合约，支持用户存入代币获得收益，支持锁仓期设置
```

## 执行步骤

1. 调用 `pm-agent`：分析需求，输出 PRD 和任务列表
2. 等待用户确认 PRD 方向（可选）
3. 调用 `architect-agent`：设计合约架构，输出 ADR
4. 调用 `contract-dev-agent`：实现合约代码
5. 调用 `security-agent`：执行安全审计
6. 如审计通过，调用 `tester-agent`：编写并运行测试
7. 输出完整的功能开发摘要

## 并行执行策略

- 步骤 1-4 串行执行（存在依赖关系）
- 步骤 5-6 可在安全审计通过后并行处理多个合约的测试
